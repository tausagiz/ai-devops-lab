# Code of Conduct

## Purpose

This repository is maintained as a personal practice and learning sandbox.
Public interaction is welcome in the form of respectful feedback, questions, and discussion.

## Expected Behavior

- Be respectful and constructive in issues and discussions.
- Focus feedback on ideas and implementation details, not people.
- Assume good intent, ask clarifying questions, and keep comments actionable.
- Use clear, concise language and keep suggestions proportional to project scope.

## Unacceptable Behavior

- Harassment, insults, discrimination, or personal attacks.
- Sexualized language or unwanted sexual attention.
- Doxxing, threats, intimidation, or sustained disruption.
- Spam, trolling, or intentionally derailing technical discussion.

## Scope

This Code of Conduct applies to all project spaces: issues, discussions, and review comments.

## Enforcement

Project maintainers are responsible for enforcement and may edit/remove content, lock threads, or block users when needed.

For serious or sensitive incidents, use GitHub reporting tools or contact a maintainer privately.

## Project-Specific Guidance

This repository is discussion-first: feedback and ideas are welcome here.
If you want to extend implementation directly, you are encouraged to use your own fork according to the license terms.

Behavior expectations live here; workflow/process rules live in [AGENTS.md](AGENTS.md).
Please treat [AGENTS.md](AGENTS.md) as the single source of truth for agent-facing workflow procedures.
